      Description

      Right now, in order to get the plan types, we depend on the account service. So, every time, we received a limit request, we made a call to Account Service. This was very resource intensive. As a result, we decided to cache this value from Account service.
      But, in cases where a customer bumps up the plan type or degrades the plan type, the only way we can get the new values is by restarting the server. This is not a good implementation.

      Therefore, the idea is to create a TTL or implement a centralized cache to do this.
